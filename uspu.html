Unique Selling Points

An AI-powered mock interview platform built for Vietnamese job seekers. Structured discovery of who uses it, what they need, and why it wins.

01

Unique Selling Points

USP 01
Job Readiness Index (JRI)
A single evolving score that tracks a candidate's readiness over time — not just a one-off grade. No competitor in the Vietnamese market offers a longitudinal, composite readiness metric.
USP 02
CV + JD Personalization
Every interview is generated from the candidate's actual CV and the specific job description they're targeting. Generic question banks are replaced by context-aware, role-specific sessions.
USP 03
Built for the Vietnamese Market
VND-denominated pricing (free → 100K → 299K/month), simulations tailored to FPT, VNG, Viettel, Momo, and Grab Vietnam — not a global product translated to Vietnamese.
USP 04
Integrated Coding Round
A full LeetCode-style coding environment is embedded directly in the interview flow — not a separate tool. Candidates practice both behavioral and technical rounds in one unbroken session.
USP 05
AI Tutor Mode (Learn Mode)
Find you are not ready? Toogle Mentor mode on and you will be guided to learn, offering Socratic hints without giving away the solution. Supports learning, not just grading.
USP 06
Radar-Based Skill Gap Analysis
Multi-dimensional readiness across Technical Skills, Communication, Problem Solving, Attitude, and Cultural Fit — visualized as a radar chart comparing current vs. target for each role.
USP 07
University Partnership Channel
A dedicated portal for career centers to track student cohort readiness at scale — opening a B2B revenue channel alongside the direct-to-candidate model.
USP 08
24/7 Availability, No Scheduling
Replaces the traditional mock interview — expensive, hard to book, and inconsistent. Candidates practice at 2am before a big interview day without coordinating anyone else's calendar.

User Stories

Persona — Fresh Graduate / Student
US-01
As a final-year computer science student, I want to upload my CV and a job description so the platform can simulate an interview specifically matched to that role, so that I stop practicing generic questions that never appear in real interviews.
PriorityCritical
Page/setup, /interview
Acceptance Criteria
  • CV (PDF/Docx) and JD file are both accepted by the upload dropzone
  • AI extraction completes and returns name, target role, tech stack, and ATS match %
  • Interview questions generated reflect skills listed in the JD, not generic prompts
US-02
As a student with no previous interview experience, I want to see a Job Readiness Index score after each session so I can track whether my preparation is actually improving week over week.
PriorityCritical
Page/dashboard, /analytics
Acceptance Criteria
  • JDI score (0–100) is shown on the dashboard after every completed session
  • Score history chart shows trend across at least 3 sessions
  • Score changes are explained with a one-line AI summary of what improved or declined
US-03
As a student preparing for a technical internship, I want to attempt a live coding challenge inside the platform so that I can practice algorithms in the same session as my behavioral interview, mirroring real hiring pipelines.
PriorityCritical
Page/coding, /coding-results
Acceptance Criteria
  • Monaco editor loads with the problem statement, examples, and constraints
  • Run button executes code and returns real test case pass/fail results
  • Submit triggers AI analysis scoring correctness, optimization, and code quality
US-04
As a student who gets nervous in high-pressure coding sessions, I want a "Learn Mode" toggle that gives me Socratic hints on the coding challenge without revealing the answer, so I can build genuine problem-solving intuition rather than just memorizing solutions.
PriorityHigh
Page/coding
Acceptance Criteria
  • Learn Mode toggle is visible and clearly labeled in the coding top bar
  • "Get AI Coach Help" returns a hint that guides without giving the solution
  • Learn Mode sessions are flagged separately in analytics (not mixed with assessment scores)
Persona — Fresh Graduate Targeting Specific Companies
US-05
As a fresh graduate applying to FPT Software, I want a company-specific interview simulation that mirrors FPT's known interview format, so that I'm not caught off guard by their specific style of technical and HR screening.
PriorityHigh
PhaseP4 — Platform
Acceptance Criteria
  • Company pack is selectable from the setup screen alongside or instead of a custom JD
  • Questions and evaluation rubrics are adapted to the target company's known format
  • Analytics report highlights gaps specific to that company's requirements
US-06
As a fresh graduate on a tight budget, I want to try the full interview flow for free before committing to a paid plan, so I can evaluate whether the platform genuinely improves my readiness before spending money.
PriorityCritical
Page/pricing, /dashboard
Acceptance Criteria
  • Freemium tier allows at least 1 complete interview session (behavioral + coding) per month
  • Upgrade prompt appears contextually — after session results, not mid-session
  • Pricing page clearly shows VND amounts and what each tier unlocks
Persona — Career Switcher
US-07
As a career switcher entering tech from a non-CS background, I want to see a radar chart comparing my current skills against the target role's requirements, so I know exactly which dimensions to close before applying.
PriorityHigh
Page/dashboard, /analytics
Acceptance Criteria
  • Radar chart shows both "current" and "target" overlaid on 5 axes
  • Action plan table maps skill gaps to specific, prioritized learning recommendations
  • Each recommendation shows a predicted JDI boost if completed
US-08
As a career switcher who struggles to articulate past experience, I want post-session feedback that breaks down each interview answer by strengths, areas to improve, and an optimal answer suggestion, so I can learn exactly how to reframe my background.
PriorityCritical
Page/analytics
Acceptance Criteria
  • Analytics accordion shows every question with the user's recorded answer
  • Each entry has three labelled sections: Strengths, Improve, Optimal Answer
  • Optimal answer reflects the role and company context, not a generic template
Persona — University Career Center Staff
US-09
As a career center advisor, I want to view cohort-level readiness metrics across all students using the platform, so I can identify which graduating classes or departments need targeted intervention before internship season.
PriorityMedium
PhaseP4 — University Portal
Acceptance Criteria
  • University dashboard shows aggregate JDI distribution across enrolled students
  • Filters available by department, cohort year, and target company type
  • Individual student reports are accessible with student's consent
US-10
As a career advisor reviewing a student's application, I want to share a student's analytics report directly with a hiring partner via a secure link, so that a verified readiness score accompanies their CV submission.
PriorityMedium
Page/analytics — Share feature
Acceptance Criteria
  • "Share with University" generates a time-limited, read-only link to the analytics report
  • Shared view shows JDI score, radar chart, and action plan — not raw answer transcripts
  • Student must explicitly consent before any advisor can generate a share link
Persona — Platform Admin
US-11
As a platform admin, I want a dashboard showing total active users, session counts, revenue by plan tier, and flagged activity, so I can monitor platform health and catch abuse without needing to query the database manually.
PriorityHigh
Page/admin
Acceptance Criteria
  • Admin page shows user count, session count, and revenue metrics at a glance
  • Session log is searchable and filterable by user, date, and status
  • Admin routes are protected and inaccessible to regular users
